# Decision Log

This document is the single source of truth for **why** non-trivial implementation
decisions were made in the Blitzy Slack proof-of-concept. Per the project's
Explainability rule, rationale MUST live here — **not** in code comments. A decision
is "non-trivial" if a competent engineer could reasonably have chosen differently.
Any deviation from a literal or obvious interpretation of the requirements MUST have
an explicit entry here; unexplained deviations are treated as defects.

Downstream code-generation agents MUST append a row to the relevant table below for
every additional non-trivial decision they make.

## Root workspace & tooling

| What was decided                                                                | What alternatives existed                                                                                 | Why this choice was made                                                                                                                                                                                                                       | What risks it carries                                                                                                                               |
| ------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| pnpm workspace declared with the `packages/*` wildcard in `pnpm-workspace.yaml` | Enumerate each package path explicitly (`packages/db`, `packages/shared`, `packages/api`, `packages/web`) | The wildcard is the AAP/checkpoint-specified workspace contract and automatically includes any future package added under `packages/` without editing the manifest. Explicit enumeration silently omits new packages and drifts from the spec. | A stray non-package directory under `packages/` would be matched by the glob; acceptable because that tree holds only the four intended workspaces. |

| The error event constant is exported as `ERROR = 'error'` (no `SOCKET_ERROR` alias)                                                                                                                                                                            | Keep the prior `SOCKET_ERROR` name, or export both                                                   | The checkpoint/shared contract specifies `ERROR`; socket typings and imports are generated against that name. | `PresenceState` modelled as a string-literal union (`'online' \| 'away' \| 'offline'`)                                                                                                                                                                         | A TypeScript `enum`                                                                                  | Same reasoning as the event constants: a union of string literals is zero-runtime, serialises directly over the wire/JSON, and narrows cleanly.                                                                                                                                                         | Values are bare strings, so a typo is caught only where the `PresenceState` type is applied; usage sites are typed, so this is contained.               |
